OpenShift Pipelines and Minio

Event Listeners

  1. Login to your OpenShift console.

  2. Select rhsso.

  3. Login with your username and password.

  4. You should see the OpenShift Web console with the summit_project

  5. Scroll down and expand the Pipelines section on the left menu. Select the Triggers. Note that we have an event listener named simple-pipeline-listener setup.

  6. Click on Trigger Templates → simple-template → YAML. The event listener is configured to run this trigger template when activated. This template runs our start-dsp-pipeline.

  7. Click on Pipelines. You should see the start-dsp-pipeline. This OpenShift pipeline is responsible for starting our OpenShift AI Data Science Pipeline.

Minio

We’re using Mino for our s3 buckets.

  1. Get the route for Mino by going to the OpenShift web console and make sure you change your Project to All Projects. Select Networking → Routes. Type minio-ui into the search bar and click the Location URL to open the Minio web console.

  2. Login to the Mino web console with the user minio and password minio123.

  3. We’ll create an event destination so that our event listener can trigger our OpenShift pipeline whenever we upload files to the s3 bucket.

    Click on Events on the left menu and click on the Add event destination button.

  4. Scroll down until you see the webhook function. Click on Webhook.

  5. We’ll need to get our OpenShift token before we can completely configure the event destination in Minio.

  6. Enter in dsp-event-listener for the Identifier, the event listener route URL for the Endpoint, and your OpenShift token in the Auth Token field.

    Click Save Event Destination

  7. Under Administration on the left hand menu click Buckets. Open the upload-files bucket and click Events for the upload-files bucket. Click on Subscribe to Event.

  8. Select the dsp-event-listener webhook from the dropdown. Select the PUT - Object Uploaded event and click Save.

Your s3 bucket is now configured to send an event to your OpenShift event listener every time a file is uploaded. In the next section we’ll upload a file and see how it runs our pipelines!
